# 照片管理工具 设计文档

## 一、产品概述

### 1.1 产品定位

一个基于Lightroom评分的本地照片批量管理工具，解决"Lightroom打完分后如何批量处理文件"的痛点。

### 1.2 核心价值

- **批量操作**：按评分一次性处理大量照片，而非逐张手动
- **安全删除**：先移到待删除目录，确认后再真删
- **智能筛选**：多维度筛选（评分/文件类型/日期/位置）

### 1.3 目标用户

- 摄影爱好者，用Lightroom管理照片
- 有大量RAW文件需要整理
- 不想手动操作数千张照片

## 四、数据模型

### 4.1 照片实体

```python
Photo {
  id: int                    # Lightroom内部ID
  file_id: int              # AgLibraryFile.id_local
  filename: str             # 文件名（DSC00121.ARW）
  rating: int               # 评分（0-5，0表示未评分）
  capture_date: datetime    # 拍摄日期（从EXIF读取）
  file_path: str            # 完整文件路径
  file_type: str            # ARW/JPG/DNG/CR2/NEF
  file_size: int            # 文件大小（字节）
  is_raw: bool              # 是否RAW格式
  # 可扩展：color_label, pick_flag, keywords等
}
```

### 4.2 数据库表关系（关键）

```
Adobe_images          # 评分/颜色标签/星级主表
  └─ rootFile → AgLibraryFile.id_local

AgLibraryFile         # 文件基础信息
  ├─ folder → AgLibraryFolder.id_local
  ├─ baseName         # 文件名不含后缀（用于配对匹配）
  └─ extension        # 文件后缀

AgLibraryFolder       # 文件夹信息
  └─ rootFolder → AgLibraryRootFolder.id_local

AgLibraryRootFolder   # 根目录（绝对路径）
```

### 5.3 功能3：移动到指定目录

**命令**：
```bash
# 把5星照片移到精选文件夹
photo-manager move --rating "=5" --dest ~/Pictures/精选/

# 把4星以上按日期自动归档
photo-manager move --rating ">=4" --dest ~/Pictures/精选/ --by-date

# 结果：~/Pictures/精选/2023/05/ 和 ~/Pictures/精选/2024/06/
```

**逻辑**：
1. 查询符合条件的照片
2. 检查目标目录是否存在，不存在则创建
3. 逐个移动文件（使用 shutil.move）
4. 生成操作日志
5. （可选）更新Lightroom目录中的路径（需要关闭Lightroom）

**输出**：
```
移动操作完成
====================
移动数量: 170张
目标目录: /Users/canghai/Pictures/精选/

移动明细:
  DSC00121.ARW → /Users/canghai/Pictures/精选/2023/05/DSC00121.ARW
  DSC00147.ARW → /Users/canghai/Pictures/精选/2023/05/DSC00147.ARW
  ...

Lightroom目录更新: 已提示重新同步
```

---

### 5.4 功能4：安全删除

**命令**：
```bash
# 把1-2星移到待删除目录（不真删）
photo-manager to-delete --rating "1..2" --dest ~/Pictures/待删除/
```

**逻辑**：
1. 移动到待删除目录
2. 生成索引文件（记录原始路径）
3. 可以通过 `restore` 命令恢复

**恢复命令**：
```bash
photo-manager restore --source ~/Pictures/待删除/
```

### 6.4 GUI方案对比

| 维度 | PyQt | Web + PyWebView |
|------|------|-----------------|
| **技术栈** | PyQt6 | FastAPI + HTML/JS + PyWebView |
| **学习曲线** | 需要学习Qt框架（信号/槽/布局） | 需要学习Web技术（但用户已有React基础） |
| **界面效果** | 原生桌面风格 | 现代Web风格 |
| **代码复用** | 100%（直接导入core模块） | 100%（FastAPI导入core模块） |
| **跨平台** | ✅ Mac/Win/Linux | ✅ Mac/Win/Linux（打包后） |
| **依赖** | PyQt6（~40MB） | FastAPI + PyWebView（~10MB） |
| **开发效率** | 需要手写UI代码 | HTML/CSS开发界面更快 |
| **未来扩展** | 难以Web化 | 容易Web化（直接用浏览器） |
| **调试** | 需要运行桌面应用 | 浏览器开发者工具 |

### 6.5 GUI方案推荐

**v2.0阶段两者都支持，用户可根据需求选择：**

- **选PyQt**：想要原生桌面体验，不想学Web技术
- **选Web+PyWebView**：想要现代界面，已有Web技术基础，未来可能Web化

**核心逻辑在 `src/core/` 下，两种GUI都直接调用，代码100%复用。**

```yaml
        python-version: ['3.10', '3.11']
    
    steps:
    - uses: actions/checkout@v4
    
    - name: Set up Python ${{ matrix.python-version }}

      with:
        python-version: ${{ matrix.python-version }}
    
    - name: Install dependencies
      run: |
        python -m pip install --upgrade pip
        pip install pytest pytest-cov
    
    - name: Run tests
      run: |
        pytest tests/ --cov=src --cov-report=xml
    
    - name: Upload to Codecov
      uses: codecov/codecov-action@v3
      with:
        files: ./coverage.xml
```

### 11.7 遇到问题怎么办

**编译/运行错误：**
1. 仔细看报错信息
2. Google报错 + 项目名
3. 查看GitHub issue（如果是第三方库）
4. 在commit message里记录问题和解决方案

**测试失败：**
1. 看失败的是哪个测试
2. 添加 `--pdb` 进入调试
3. 或在测试里加 `print` 打印中间状态
4. 修复后重新测试

**Git冲突：**
1. `git pull --rebase origin develop`
2. 手动解决冲突
3. `git add .` 标记解决
4. `git rebase --continue`
5. push前确保能跑通测试
